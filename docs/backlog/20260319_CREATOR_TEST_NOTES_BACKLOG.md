# 2026-03-19 제작자 사용 테스트 메모 백로그

## 목적

실제 제작 흐름을 사용하면서 나온 메모를 흩어진 의견이 아니라,
다음 구현 우선순위를 잡을 수 있는 백로그 형태로 정리한다.

이 문서는 `바로 수정할 버그 의심`, `제품 결정이 필요한 항목`, `메이커 UI/UX 개선` 을 구분해 적는다.

## 정리 기준

- `P1`: 기능 오작동 가능성, 저장/진행/검증처럼 사용을 막는 항목
- `P2`: 작업 흐름을 방해하는 편집 UX 문제
- `P3`: 제품 정책이나 표현 방식을 더 결정해야 하는 항목
- `재현 필요`: 아직 확정 버그로 단정하지 않고 확인이 필요한 항목

## P1. 우선 확인 / 수정 후보

### 1. 필수 입력 누락 시 피드백 부족

- 현상:
  - 태그 같은 필수 정보를 빼먹었을 때 왜 저장/다음이 안 되는지 즉시 이해하기 어렵다.
- 상태:
  - `1차 수정`
- 필요한 방향:
  - 저장 또는 다음 버튼 클릭 시 막힌 이유를 토스트/팝업/인라인 메시지로 알려준다.
  - 첫 오류 필드로 자동 스크롤 이동한다.
  - 해당 섹션을 시각적으로 강조한다.
- 2026-03-24 1차 수정:
  - 현재 스텝 검증 이슈 패널 추가
  - 스텝 이동은 막지 않고, 현재 스텝 안에서 `확인 항목` 패널로만 안내
  - 이슈 패널의 `위치 보기`로 첫 수정 위치까지 바로 이동
  - Step 3/5처럼 내부 탭이 있는 화면은 관련 탭을 자동으로 열고 이동
  - 플레이어 수 불일치는 Step 1이 아니라 Step 3(플레이어) 쪽에서만 안내
- 남은 확인:
  - `위치 보기` 스크롤이 목표 섹션보다 약간 아래로 떨어지는 오프셋 보정
  - 현재 위치 강조가 더 필요한지

### 2. 게임별 세션 분리 의심

- 현상:
  - 다른 곳에서 다시 들어가도 진행 중인 세션으로만 들어가는 것처럼 보인다는 메모가 있었다.
- 상태:
  - `원인 확인`
- 확인 포인트:
  - `/play/[gameId]` 진입 시 활성 세션 목록의 첫 번째 세션을 자동 선택하고 있었다.
  - 같은 게임에서 새 세션을 명시적으로 시작하거나 다른 활성 세션을 고르는 UX가 부족했다.
- 2026-03-23 1차 수정:
  - GM 화면에 활성 세션 목록을 노출
  - `새 세션 시작` 버튼을 세션이 있어도 항상 노출
  - 세션별 코드/페이즈/참가 인원 요약 표시
- 남은 확인:
  - 실제 사용 흐름에서 세션 전환 UX가 충분히 이해되는지
  - 다른 활성 세션에 너무 쉽게 들어가 진행할 수 있는 운영 위험을 어떻게 줄일지
- 우선순위:
  - 높음

### 3. 제작도우미 응답 잘림 / 이전 입력 잔류 의심

- 현상:
  - LLM 응답이 쓰다가 잘리거나, 이전 채팅 흔적이 어색하게 남는 느낌이 있었다.
- 상태:
  - `고확률 원인 확인`
- 확인 포인트:
  - 현재 응답 토큰 상한이 낮아 긴 문안 응답에서 `max_output_tokens` 잘림 가능성이 있었다.
  - 중간 종료 시 부분 응답을 그대로 쓰지 않고 재시도/오류 처리해야 한다.
  - 긴 응답에서 카드 렌더링이 잘리는지
  - 실패한 메시지/부분 응답이 목록에 남는지
- 2026-03-23 1차 수정:
  - 기본 응답 토큰 상한 상향
  - `max_output_tokens` 로 잘리면 자동 재시도
  - 재시도 후에도 잘리면 부분 응답 대신 명시적 오류 표시
- 우선순위:
  - 높음

## P2. 메이커 편집 UX 개선

### 1. 범인 설정 UX 재정리

- 메모:
  - 범인 설정은 결국 플레이어 정보에서 하는 편이 자연스럽다.
  - 현재 앞쪽 설정 구간과 역할이 겹치거나 충돌할 수 있다.
- 상태:
  - `1차 수정`
- 필요한 방향:
  - 범인 지정의 단일 진입점을 정한다.
  - 플레이어 카드와 스토리 설정 사이 책임을 정리한다.
- 2026-03-24 1차 수정:
  - 범인 지정 UI를 Step 2에서 제거
  - Step 3(플레이어) 상단 공용 섹션으로 이동
  - 삭제된 캐릭터가 범인으로 남아 있으면 자동 해제

### 2. 이미지 업로드 시 편집 화면이 지나치게 길어짐

- 메모:
  - 이미지가 늘어나면서 편집 화면 길이가 과도하게 길어진다.
- 개선 후보:
  - 업로드 여부만 작은 상태 배지/체크로 먼저 표시
  - 썸네일은 접힘 상태로 두고 클릭 시만 확장
  - 미리보기 크기를 더 줄이거나 고정 높이 카드로 제한
- 우선순위:
  - 높음

### 3. 이미지 URL 입력과 업로드 UI 일관성 부족

- 메모:
  - URL 입력과 업로드 UI가 아직 서로 튀는 느낌이 있다.
  - 라운드별 이미지는 아직 업로드 없이 URL만 있다.
- 필요한 방향:
  - 이미지 필드 공통 규칙을 정리한다.
  - 업로드/URL/미리보기/제거 구조를 라운드 이미지까지 통일할지 결정한다.
- 우선순위:
  - 높음

### 4. 장소/단서 편집 화면의 시각적 구분 부족

- 메모:
  - 장소/단서는 세로로 길어질 수밖에 없는데, 블록 간 구분이 약해서 현재 위치를 찾기 어렵다.
- 개선 후보:
  - 장소 카드와 단서 카드의 배경 톤 분리
  - 접기/펼치기
  - 현재 편집 중인 블록 강조
  - 모바일/좁은 화면에서 카드 레이아웃 재정리
- 우선순위:
  - 높음

### 5. 플레이어 연관 단서 편집 가독성 부족

- 메모:
  - 단서가 많아지면 전체 리스트에서 “자기 관련 단서”를 고르기 어렵다.
  - `물적 증거` 같은 타입 표기보다 장소가 더 유용할 수 있다.
  - 본인 위치/맥락도 수동 입력 부담이 크다.
- 개선 후보:
  - 단서 선택 목록에 `장소명` 표시
  - 타입보다 장소/소유자 중심 보조 정보 노출
  - 검색/필터
  - 선택 후 자동 설명 템플릿 제안
- 우선순위:
  - 중간

### 6. 표지 이미지 위치와 썸네일 노출 조정

- 메모:
  - 표지 이미지 박스 위치와 크기를 조금 더 다듬을 필요가 있다.
  - 제작기 안에서 보이는 썸네일 배치도 재검토 필요.
- 우선순위:
  - 중간

## P3. 제품 결정 / 정책 결정 필요

### 1. GM 세션 진입/전환 보호 수준

- 메모:
  - 여러 세션 운용은 가능하지만, GM이 다른 활성 세션에 너무 쉽게 들어가 진행을 바꿔버릴 수 있다는 우려가 있다.
- 결정 포인트:
  - 현재처럼 같은 게임의 활성 세션을 자유롭게 전환할지
  - 별도 세션 선택 화면/퍼널로 분리할지
  - 진행 중 세션 진입 전에 확인 모달이 필요한지
  - 이미 참가자가 있는 세션에는 경고를 보여줄지
- 우선순위:
  - 중간

### 2. 장소별 이미지 필요 범위

- 메모:
  - 모든 장소에 이미지를 둘 필요는 없어 보인다.
  - 이벤트 장소 정도만 필요한지 고민이 필요하다.
- 결정 포인트:
  - 장소 대표 이미지가 기본 기능인지
  - 특정 라운드/이벤트 장소 이미지가 더 중요한지
  - 장소 이미지 없이도 플레이 UX가 충분한지

### 3. 단서 제안 빠른 액션의 입력 맥락 강화

- 메모:
  - `단서 제안` 딸깍 시 어느 장소인지, 누구와 관련된 단서인지 같이 볼 수 있으면 좋겠다.
  - 현재 프롬프트가 3개 제안으로 고정된 것도 재검토 필요하다.
- 개선 후보:
  - 현재 선택한 장소/플레이어 기준 제안
  - 출력 개수 선택
  - 장소용 / 인물용 / 반전용 제안 분리
- 우선순위:
  - 중간
