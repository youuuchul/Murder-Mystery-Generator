## 오늘 완료된 핵심 작업

### 플레이어 재접속 / UI 정리

- GM `재참가 허용` 시 슬롯만 해제하고, 캐릭터의 진행 상태는 유지하도록 수정
- 재참가 시 기존 `playerState`, 인벤토리, 라운드 진행, 투표 상태를 새 토큰으로 이어받도록 수정
- 플레이어 화면 `내 정보` 순서를 `이미지 → 배경 → 상세 스토리 → 비밀/반전 → 승점 조건 → 관련 단서`로 재정렬
- `승점 조건`, `관련 단서`도 접고 펼칠 수 있게 변경
- 라운드 시작 시 플레이어 탭이 자동으로 `장소 탐색`으로 이동되게 수정

### AI 제작도우미 출력 모드 1차 분리

- `chat` 요청에 대해 `guide / draft` 응답 모드 분리
- 입력창에 `자동 / 가이드 / 문안` 모드 토글 추가
- `자동` 모드는 질문 키워드 기준으로 서버에서 `guide / draft` 추론
- `draft` 응답은 `title / body / notes` 구조로 파싱하고, UI에서 붙여넣기용 본문 카드로 별도 렌더
- 빠른 액션은 기존처럼 `guide` 응답으로 유지
- assistant step 번호 범위를 `1~6` 으로 정리해서 엔딩 step도 정상 참조

### AI 제작도우미 안정화 / UX 보정

- 후속 채팅에서 `previous_response_id` 에 의존하지 않도록 수정
- 최근 대화 8턴을 직접 prompt에 포함해 후속 질문 맥락이 유지되게 변경
- 실패한 요청의 사용자 메시지는 채팅 목록에서 되돌려 에러 턴이 문맥을 오염시키지 않게 정리
- `draft` 문안을 `서사형 산문 / 설명형 텍스트 / GM 진행문` 프로필로 나눠서 프롬프트 강화
- 스토리형 요청에서는 플레이어 수, 단서 개수, 타임라인 시각 같은 설계 메타를 본문에 그대로 쓰지 않도록 금지 규칙 추가
- 메이커 우하단 AI 런처가 하단 저장 액션바와 겹치지 않도록 위치 계산 수정
- 런처 보조 문구를 `초안 작성 · 점검 · 제안` 으로 갱신
