# 2026-03-19 로컬 제작자 유저 테스트 계획

## 목적

배포 전 단계에서 현재 로컬 버전으로 실제 제작자 테스트를 진행할 때,
접속 방법, 역할, 주소, 저장 위치, 주의사항을 한 문서로 정리한다.

이 문서는 “지금 상태로 무엇을 테스트할 수 있는가” 에 초점을 둔다.

## 테스트 전제

- 서버 원본은 호스트 PC 1대다.
- 저장 데이터는 모두 호스트 PC의 로컬 `data/` 디렉토리에 기록된다.
- 아직 권한 분리가 없으므로, 테스트 참여자는 URL만 알면 다른 게임도 수정 가능하다.
- 따라서 같은 게임 동시 편집은 피하고, 참여자별 담당 게임을 나눠서 테스트하는 편이 안전하다.

## 기본 테스트 흐름

1. 호스트가 로컬 서버를 실행한다.
2. 호스트가 라이브러리 접속 주소를 공유한다.
3. 참여자는 라이브러리에서 새 게임을 만들거나 지정된 게임을 편집한다.
4. 저장 시 결과는 호스트 PC의 `data/games/{gameId}` 아래에 기록된다.
5. 필요하면 호스트가 해당 게임으로 세션을 시작해 플레이 흐름까지 이어서 테스트한다.

## 실행 방법

### 같은 Wi-Fi / LAN 환경

호스트:

```bash
npm run dev
```

공유 주소 예시:

- 라이브러리: `http://<HOST-LAN-IP>:3000/library`
- 참가 진입: `http://<HOST-LAN-IP>:3000/join`

### 다른 네트워크 / 원격 기기 테스트

호스트:

```bash
npm run dev:tunnel
```

공유 주소 예시:

- 라이브러리: `<tunnel-url>/library`
- 참가 진입: `<tunnel-url>/join`

## 참여자에게 전달할 안내 문구 예시

### 제작자 테스트용

- 접속 주소: `/library`
- 할 일:
  - 새 게임 생성 또는 지정 게임 편집
  - Step 1~6 저장 흐름 확인
  - 이미지 업로드 / 소개글 / 스크립트 / 엔딩 작성 흐름 확인
  - AI 제작도우미 문안/가이드 응답 확인
- 주의:
  - 다른 사람이 만지는 게임은 열지 않기
  - 같은 게임 동시 수정 피하기

### 플레이 테스트용

- 접속 주소: `/join`
- 할 일:
  - 코드 입력
  - 캐릭터 선택
  - 장소 탐색 / 인벤토리 / 투표 흐름 확인

## 저장 위치

현재 저장 위치:

- 게임 패키지: `data/games/{gameId}/game.json`
- 라이브러리 메타: `data/games/{gameId}/metadata.json`
- 업로드 자산: `data/games/{gameId}/assets/*`
- 세션 상태: `data/sessions/{sessionId}.json`

즉 테스트 참여자가 저장해도, 실제 데이터는 모두 호스트 로컬 디스크에 쌓인다.

## 테스트 체크리스트

### 제작

- 라이브러리에서 새 게임 생성 가능
- Step 1~6 이동과 저장 상태가 이해되는지
- 소개글과 표지 카드가 라이브러리에서 잘 보이는지
- 이미지 업로드/제거/교체가 자연스러운지
- AI 제작도우미 `자동 / 가이드 / 문안` 모드가 구분되는지

### 플레이

- 세션 생성과 코드 참가가 잘 되는지
- 재접속 자동 복구 / 이름 기반 복구가 이해되는지
- 플레이어 화면 탭 흐름이 자연스러운지
- 투표 / 엔딩 / 결과 노출이 어색하지 않은지

## 운영 주의사항

### 1. 같은 게임 동시 편집 금지

현재는 로컬 JSON 파일 직접 저장 구조라서,
같은 게임을 여러 명이 동시에 수정하면 마지막 저장이 이전 내용을 덮어쓸 수 있다.

### 2. 테스트 전후 백업 권장

최소 백업 대상:

- `data/games/`
- 필요 시 `data/sessions/`

### 3. 테스트용 역할 분리 권장

안전한 운영 예시:

- A: 새 게임 제작 흐름
- B: 기존 게임 수정 흐름
- C: 플레이 참가 흐름

## 권장 테스트 순서

1. 호스트가 서버 실행
2. 참여자별 담당 게임/역할 배정
3. 제작자 테스트 진행
4. 테스트 종료 후 `data/games` 백업
5. 필요하면 이어서 실제 플레이 테스트
