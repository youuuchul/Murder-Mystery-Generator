# 화면 스크린샷 저장소

Murder Mystery Generator 각 화면의 실제 UI 캡처를 모아두는 폴더. 이미지 파일 자체는 **git에 올리지 않는다** (용량 누적 회피). 공유는 Google Drive + Google Sheets 임베드로 한다.

---

## 파일명 규칙

**기본 규칙**: 파일명 앞머리는 `docs/screens.json`의 `id` 값과 **정확히 일치**해야 한다. PM이 Sheet에서 ID 보고 파일을 바로 찾는다.

```
P-XXX.png                     # 해당 화면의 대표 이미지 1장
P-XXX_<variant>.png           # 상태 변형 (선택, 필요시)
P-XXX_mobile.png              # 모바일 뷰포트 명시 구분 (선택)
```

### 예시

| 파일명 | 의미 |
|--------|------|
| `P-017.png` | 플레이어 메인 화면 대표 (탭 기본 상태) |
| `P-017.3.png` | 인벤토리 탭 기본 상태 |
| `P-017.3_modal.png` | 인벤토리 탭 + CardDetailModal 열린 상태 |
| `P-017.4_acquire.png` | 장소 탐색 탭 + 단서 획득 직후 토스트 |
| `P-014_empty.png` | 조인 페이지 + 빈 슬롯 상태 |
| `P-014_inherit.png` | 조인 페이지 + "이어받기 가능" 배지 노출 |
| `P-007.5_upload.png` | 메이커 Step5 + 라운드 이미지 업로드 후 |

### 피해야 할 네이밍
- ❌ `main-screen.png` (ID 없음 → 시트 연결 안 됨)
- ❌ `P17-3.png` (ID 형식 불일치, 대시 오용)
- ❌ `P-017_v2.png` (버전 넘버 금지 — 항상 최신만 유지)

## 권장 캡처 뷰포트

| 화면 카테고리 | 뷰포트 | 이유 |
|---------------|--------|------|
| 플레이어 화면 (`P-014`, `P-017.*`) | **모바일** 375x812 기준 | 실제 사용 기기 |
| 메이커/관리 (`P-007.*`, `P-009~P-011`) | **데스크톱** 1440x900 기준 | 편집 UI는 데스크톱 주 사용 |
| 가이드/홈/라이브러리 | 모바일 + 데스크톱 둘 다 (선택) | 반응형 확인 |

## 캡처 요령
- 최신 `main` 브랜치 기준 production 상태 우선
- 개인 식별 정보(실명/이메일) 가린 상태로 찍기
- 상태 변형(`_modal`, `_empty` 등)은 **PM/디자인 관점에서 의미 있는 차이만** — 모든 상태를 남길 필요 없음

## 시트/Drive 연동

### 현재 (Phase 1)
- `npm run sync:screens` 실행 시 이 폴더의 파일 존재 여부가 Sheet "스크린샷" 컬럼에 표시됨 (`있음 / 미캡처`)

### 예정 (Phase 2)
- `npm run sync:screenshots` 실행 시:
  1. 이 폴더의 모든 `*.png`를 Google Drive `화면 설계서/스크린샷/` 폴더에 업로드
  2. 공유 링크 획득
  3. Sheet의 해당 ID 행에 `=IMAGE("https://drive.google.com/uc?id=<id>")` 수식 자동 삽입
- Phase 2 자동화 시 필요한 추가 환경변수:
  - `GOOGLE_DRIVE_FOLDER_ID` (업로드 대상 Drive 폴더 ID)
- 구현 백로그: `docs/STATUS.md` "기획·설계 인프라" 섹션 참조.

## FAQ

**Q. 그냥 Drive에 직접 올리면 안 되나?**
- 가능. 그 경우 파일명만 규칙(`P-XXX.png`) 지키면 시트 자동 임베드는 Phase 2 스크립트가 처리.

**Q. 여러 상태를 모두 찍어야 하나?**
- 아니. 화면당 대표 1장이 기본. 중요한 상태 변형만 추가.

**Q. 스크린샷 업데이트 주기는?**
- 대형 UI 리뉴얼 시 일괄 갱신. 작은 수정에는 갱신 불필요.
